1. <h1> - <h6>
    • Use for defining headings of different levels, with <h1> being the highest level and <h6> the lowest.
    • Best practice is to reserve <h1> for the main title of the page and use other levels for subheadings.
    • Headings should reflect the hierarchical structure of the content.
  2. <p>
    • Used for defining paragraphs of text.
    • Best practice is to wrap blocks of text in <p> tags to improve readability and maintainability.
    • Avoid using multiple consecutive <p> tags for styling purposes; prefer CSS for styling.
  3. <em>
    • Used to emphasize text.
    • Best practice is to use <em> for semantic emphasis, rather than for styling purposes.
    • Screen readers may pronounce emphasized text differently, so use judiciously.
  4. <strong>
    • Used to denote strong importance, typically displayed as bold.
    • Best practice is to use <strong> for semantic importance, such as key terms or warnings, rather than for styling purposes.
    • Avoid using <strong> solely for visual styling; prefer CSS for styling.
  5. <blockquote>
    • Used for quoting blocks of text from another source.
    • Best practice is to use <blockquote> for long quotations, and <q> for shorter inline quotations.
    • Include a cite attribute to provide the source of the quotation.
